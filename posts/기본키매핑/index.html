기본키 매핑

Hyun Mar 13, 2022 2022-03-13T00:00:00+09:00
Mar 13, 2022 2022-03-13T20:37:31+09:00 1 min

@Entity, @Table
@Column
@Id, @GeneratedValue

데이터베이스 스키마 자동생성
hibernate.hbm2ddl.auto <property name=“hibernate.hbm2ddl.auto” value=“create” />

  • create -기존 테이블 drop 후 생성
  • create-drop - 생성후 종료시점에 drop
  • update - 변경만 alter
  • validate - 엔티티와 테이블 정상 매핑만 확인
  • none 운영에서는 validate나 none만 사용

@Column
@Temporal - 날짜 타입 매핑
@Enumerated -enum 타입 매핑
@Lob - BLOB, CLOB 매핑 (large object)
@Transient - 매핑 하지 않음

  • GenerationType.Identity
    auto_increment 는db에 넣어져봐야 pk 알수 있어서 commit 시점 외에 예외적으로 Persist 시점에 db쿼리 수행

  • Sequence 타입은
    모아와서 identity와 다르게 영속성 컨택스트에 넣어놓을 수 있음
    allocationSize(default = 50) - 성능최적화에 사용됨
    여러개(기본 50)개까지 미리 여러개 값을 할당받아오고
    메모리에서 그 값 안에서 여러 객체에 50개까지 시퀀스 조회 쿼리 안날리고 pk할당해주다가 50개 다 쓰면 다시 시퀀스 얻어오도록
    -숫자 구멍은 생길수 있음

GenerationType.Table은 사실 잘 사용안함
성능이슈, 테이블이기 때문에 락걸릴 수도 있음
